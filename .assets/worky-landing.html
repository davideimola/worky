How it works

From code to classroom
in three steps.

Worky handles distribution so you can focus on teaching.

01 — Create

Write your workshop

Use the Worky Go library to define steps, instructions, and automatic validators. Each step can check if the participant completed the task before unlocking the next one.

import "github.com/worky/worky"
02 — Build

Compile to one binary

Run worky build and get a single executable. Everything bundled UI, content, validators. Nothing to install on the participant's machine.

03 — Distribute

Participants just run it

Share the file. Participants download it, run one command, and their browser opens a guided interface. Step-by-step instructions, automatic validation, zero setup.

$ ./my-workshop → localhost:7331
For creators

Built for developer educators.

Every decision in Worky is made to reduce friction between you and your participants. You teach, it handles the rest.

Automatic step validation

Define validators in Go that check if participants completed each step correctly. The workshop knows when they're done, no manual confirmation needed.

Single binary distribution

One file. No Docker, no npm install, no Python venv. Participants download the binary, run it, done. Works on macOS, Linux, and Windows.

Browser-native UI

The workshop UI lives in the browser, no Electron, no GUI toolkit. Clean, accessible, customisable. Participants follow along in a familiar environment.

Go-native authoring

Write workshops in Go. No YAML, no DSL, no proprietary format. Idiomatic Go code you can version, test, and refactor like any other project.

Gated progress

Each step is locked until the previous one is validated. Keeps everyone at the right pace, no one skips ahead, no one gets left behind.

Fully customisable

The UI template is yours to adapt, colour it, brand it, extend it. Worky stays out of the way with just a discreet badge at the bottom.

Code

Looks like Go.
Because it is.

No new syntax to learn. Define your workshop the same way you'd write any Go program, clear, testable, version-controlled.

workshop.go Go
package main import ( "embed" "github.com/davideimola/worky" "github.com/davideimola/worky/checks" ) //go:embed all:site var site embed.FS func main() { worky.New(worky.Config{ Name: "Docker 101", SiteFS: site, Chapters: []worky.Chapter{{ Name: "Setup", Checks: []worky.Check{{ Run: checks.CommandSucceeds("docker", "info"), }}, }}, }).Run() }

Steps are just functions

Each step is a Go function that returns an error. Return nil and the step completes. That's the entire validation API.

Validators are composable

Use built-ins like checks.CommandSucceeds, checks.FileExists, checks.HTTPStatus — or write your own. Anything returning an error works.

One command to build

Run worky build and get a self-contained binary. Ship it with scp, a GitHub release, or a simple download link.
